## Release Notes Format

Each release has its own file: `releases/v{major}.{minor}.{patch}.md`

```markdown
# CloudStream v1.2.0 — "Spring Update"

*Released: 2026-06-15*

## Highlights

Brief paragraph on the most important changes in this release.

## What's New

- **Feature name** — description (PR #123)
- ...

## Improvements

- **Improvement** — description (PR #125)
- ...

## Bug Fixes

- Fixed crash when switching channels rapidly (PR #127)
- ...

## Known Issues

- None

## Updating

Update from the App Store or Google Play. No manual steps required.
```

---

## Versioning Policy

- **Patch releases** (1.0.**1**) — Bug fixes, security patches, no new features
- **Minor releases** (1.**1**.0) — New features, backwards compatible
- **Major releases** (**2**.0.0) — Breaking changes, significant redesigns

---

## Changelog Conventions

- Entries are sorted by feature area
- PR numbers link to the GitHub PR
- Breaking changes are marked ⚠️
- Known issues are listed honestly
